{# ============================================================================= Level 2.2 — Jinja2 Templates: Base Layout ============================================================================= QUESTIONS: 1. What is template inheritance? A base template defines the STRUCTURE (header, footer, layout). Child templates OVERRIDE specific blocks (content, title). Like OOP inheritance — DRY principle for HTML! 2. What is {% block %}? A named placeholder that child templates can fill in. {% block content %}{% endblock %} = "put page content here" 3. What is {{ }}? Output a variable's value (auto-escaped for security!). {{ book.title }} → "Clean Code" (HTML entities escaped) 4. What is {% %}? Logic/control flow: if, for, block, extends. {% for book in books %} ... {% endfor %} 5. Why does Jinja2 auto-escape? If book.title = "" Without escaping: browser executes the script! With auto-escape: displays as literal text. Safe by default. ============================================================================= #} {% block title %}MyBookShelf{% endblock %} {# Q: What does url_for('static', filename='...') do? Generates the correct URL for static files. Flask serves files from the /static/ directory automatically. In production, nginx will serve these instead (Layer 4). #}
